Начало работы

Перетащи один или несколько .txt/.csv/.log/.tsv/.dat файлов в окно либо нажми кнопку Открыть и выбери их.

  • Несколько файлов с одинаковыми тегами автоматически склеиваются во времени (индикатор ⟷ N в шапке).
  • Нажми + Файл, чтобы добавить ещё файл к текущим данным.
  • Сброс (красная кнопка) очищает все загруженные данные и выбор параметров.
Быстрый старт: перетащи файл → выбери параметры слева → выбери режим (Вместе/Раздельно) → масштабируй колёсиком или Plotly-бокс-зумом.

Сессии (💾 в шапке)

Кнопка 💾 Сессия в правом верхнем углу раскрывает выпадающее меню. Сессия — полный снимок работы: данные, выбор параметров, зум, курсоры, маркеры, уровни, заголовки графиков, тема. Хранится в IndexedDB браузера (квота обычно 50%+ свободного места на диске), так что закрыл и переоткрыл — продолжил с того же места. Большие логи (100 тыс. точек × десятки параметров) влезают без проблем.

  • Сохранить сессию… — запрашивает имя (по умолчанию имя первого файла + дата-время), пишет слепок.
  • Ниже — список сохранённых сессий, клик по имени → загрузка (с подтверждением, если сейчас есть открытые данные).
  • Справа от имени — ✕ для удаления.

Отличие от пресетов: пресет — только вид (выбор параметров, стили, зум) при тех же уже загруженных данных. Сессия — полный снимок вместе с данными; восстанавливается «с нуля» без файлов.

Несколько наборов в одной вкладке: работаешь с набором A → сохранил «Турбина пуск 2026-04» → сбросил → загрузил другой файл → сохранил «Компрессор авария» → теперь в меню два слота, переключаешься между ними мгновенно, без перезагрузки файлов и ре-зумирования.

Формат v2: точки хранятся как плоские x[]/y[] массивы вместо объектов [{ts,val},...] — размер payload сокращается примерно вдвое. Исходный текст файла в сессии не пишется — если нужна «💾 Сохранить с переименованными тегами», откройте файл заново. Старые сессии (формат v1) автоматически мигрируют при первой загрузке.

Пресеты видов

Верхняя секция сайдбара. Сохраняют текущий вид целиком и восстанавливают одним кликом:

  • Что сохраняется: выбранные параметры (SEL), режим отображения (вместе/раздельно, время/XY), сглаживание + сила, прореживание, аномалии, высота, расстояние осей, T=0, диапазон Y, слайдеры X, цвета/ширины/пунктиры линий.
  • + Сохранить — запрашивает имя, сохраняет слепок состояния.
  • Клик по имени пресета — применить.
  • ✕ — удалить.

Хранятся в localStorage под ключом loggraph_presets. Пережили бы сценарий «Пуск / Рабочий режим / Останов» — одинаковый набор параметров и зум, не надо каждый раз перенастраивать.

Параметры, поиск, сайдбар

Под пресетами — список параметров. При 50+ тегах используй поле 🔍 Поиск: фильтрует по подстроке в полном теге, short-имени, вашем кастомном имени и в единицах измерения.

Каждая строка:

  • Цветной квадрат — клик открывает пикер цвета.
  • Имя — редактируемая подпись (появится в легенде). Пусто = отображается short-имя.
  • Мета-строка — показывает полный тег, количество точек, иконку слияния ⟷, признак дискретности ⎍, имя исходного файла.
  • Стиль линии: 5 типов пунктиров (━ ╌ ··· ─· ──) и толщина − / +.
  • Уровни Lo / Hi — см. раздел «Уровни».
  • Ед: — единицы измерения, показываются над осью Y. Ввод фиксируется по Enter или клику вне поля.
  • ⇌ — кнопка конверсии (см. раздел «Единицы»).

Клик по строке — вкл/выкл отображение параметра.
Shift+клик — solo-режим: показать только этот параметр, остальные скрыть. Удобно, когда нужно быстро сфокусироваться на одном, а потом вернуться через «Все».

Кнопки Все / ⊘ над списком выделяют всё или снимают выделение (применяется к фильтрованному списку, т.е. к видимому).

Режимы отображения

  • Вместе — все параметры на одной панели, каждая со своей осью Y слева. Полезно для сравнения трендов.
  • Раздельно — каждый параметр в отдельном чарте, оси X синхронизированы при зуме.
  • Время / X-Y — время на X или любой параметр как X (поле «Ось X (параметр)»). В XY-режиме строится характеристическая кривая.
  • Склейка — соединять разрывы во времени или разбивать линию на сегменты (определяется автоматически по скачку интервалов).
  • Миникарта — лёгкий слайдер-навигатор под графиком: пустая дорожка с двумя хендлами. Тяни середину — панорамирование, тяни края — меняется ширина видимого окна. Клик по пустой области — прыжок. Работает в режимах «Вместе» и «Раздельно».

Зум и навигация

Способы масштабирования по X:

  • Колёсико — зум к точке под курсором.
  • Перетаскивание рамкой — стандартный box-zoom Plotly.
  • Слайдеры «От/До» в сайдбаре — прицельное масштабирование оси X (как обычный зум, но задаётся числом).
  • Поля чч:мм:сс под слайдерами — прицельно задать диапазон вида.
  • Миникарта под графиком — удобна для быстрого скроллинга длинного лога.
  • Двойной клик на плоте — сброс в autorange.

История зума: кнопки ◀ ▶ и Alt+← / Alt+→. Каждый зум записывается в стек (глубина 40), можно вернуться на несколько шагов назад.

Y-зум: перетаскивание рамкой по вертикали внутри области графика зумит Y (а не X). В режиме «Раздельно» — только текущего чарта; в «Вместе» — оси, на которую попал клик (Plotly выбирает ближайшую).

Ручной Y-диапазон: поля «Ось Y (раздельный режим)» в сайдбаре (Min / Max). Работает только в «Раздельно», потому что в «Вместе» у каждого параметра своя ось. Кнопка Авто Y сбрасывает.

Сохранение Y-зума при изменении набора параметров: если ты подкрутил шкалу Y у параметра А (вручную или зумом рамкой) и добавляешь/убираешь параметр B, Y-зум параметра А НЕ сбросится. Ранее эта ручная настройка терялась — теперь на слой пересборки графика сохраняются все ручные Y-диапазоны по тегам.

Производительность

Если зум «дёргается» при большом числе точек/параметров:

  • Трассы свыше 2000 точек автоматически идут через WebGL (scattergl) — на порядок быстрее SVG.
  • Миникарта — легковесный слайдер (только DOM + CSS, без пересчёта на каждый тик колёсика).
  • prepareTraceData мемоизируется LRU-кешем на 64 записи. Повторные рендеры с теми же параметрами (например, при переключении марцкеров) мгновенны.
  • При больших наборах ставь прореживание LTTB — оно сохраняет форму и при этом показывает ≤ 5000 точек.
  • Выключай Аномалии и 👁 Исход., когда они не нужны — каждая добавляет по одной трассе на параметр.
  • Для точечной работы с коротким интервалом используй слайдеры «От/До» или видимый диапазон — рендер тот же, но глаз видит меньше точек.

Сглаживание

  • Нет — исходные данные.
  • Сплайн — Plotly spline-интерполяция (меняется натяжение).
  • СС — скользящее среднее (центрированное окно).
  • EMA — экспоненциальное сглаживание (регулируется α).
  • Гаусс — свёртка с гауссовым ядром (σ).
  • Сав-Гол — Савицкий-Голей, сохраняет пики лучше СС (полином 2–3 порядка).

Кнопка 👁 Исход. показывает исходную линию поверх сглаженной (бледным пунктиром).

Слайдер Сила регулирует параметр для выбранного алгоритма.

Прореживание

При большом числе точек (> 5000) применяется прореживание для плавного рендера:

  • LTTB — Largest-Triangle-Three-Buckets, визуально наиболее точный (рекомендуется).
  • Мин-Макс — в каждой бакете берутся мин и макс, сохраняет пики.
  • Каждая N — просто каждая N-я точка (самый быстрый).

В счётчике точек сверху чарта показывается N pts (⚡M) — исходное и отрисованное.

Измерение A/B (курсоры)

Кнопка 📏 Измерение в шапке:

  • Первый клик по графику ставит курсор ▸A (голубой).
  • Второй клик — курсор ◆B (красный), включается режим диапазона.
  • Тяни курсор мышью — двигать.
  • ← / → двигают B по 1/60 диапазона.
  • + / − зумят вокруг B.
  • Esc убирает курсоры.

Снизу появляется таблица «A | B | Δ | Min↔ | Max↔ | Avg↔» для каждого выбранного параметра в диапазоне A↔B.

Маркеры (события, предупреждения, аварии, заметки)

Подсистема в сайдбаре, не путать с «Измерением». Позволяет помечать точки на оси времени типизированными маркерами:

  • ● Событие — регулярная веха (пуск, переход режима).
  • ▲ Предупр. — предупреждение, отклонение.
  • ⚠ Авария — критическое событие.
  • ℹ Заметка — произвольный комментарий.
  • ◆ Своя — пользовательский.

Как поставить: в секции «Маркеры» выбери тип в дропдауне → нажми + Поставить → кликни на графике → введи текст (можно пустой). Esc отменяет режим.

Привязка к параметру (автоматическая): если ты кликаешь рядом с линией, маркер автоматически привязывается к этому параметру. В списке ты увидишь «⟶ {имя}», а на графике к вертикали добавится цветной кружок прямо на линии параметра в точке события — очень наглядно для «здесь TNH скакнул». Клик по пустой области графика → маркер без привязки, только вертикаль.

Список маркеров под фильтрами:

  • Клик по строке — график панорамируется к этому маркеру.
  • Двойной клик — изменить текст.
  • ✕ справа — удалить.

Фильтры: цветные чипы сверху — клик включает/выключает видимость по типу (и в списке, и на графике). Поле «🔍 Поиск в тексте» фильтрует по подстроке в описании.

Экспорт:

  • ⬇ JSON — все маркеры в JSON (для передачи коллеге).
  • ⬆ JSON — импорт (добавляется к текущим).
  • CSV — журнал событий для Excel: колонки «Время / Тип / Текст».

Массовое удаление (🗑 Очистить маркеры): снести все маркеры разом с подтверждением.

Навигация клавиатурой: [ — к предыдущему, ] — к следующему видимому маркеру относительно центра текущего диапазона. Уважает активные фильтры и поиск.

Маркеры сохраняются в localStorage и переживают перезагрузку страницы. Сброс данных их не трогает — удаляй явно кнопкой «🗑 Очистить» или ✕ на строке.

Легаси: если в браузере есть аннотации из прежней версии, они автоматически мигрируют в маркеры типа ℹ Заметка при первом запуске.

Аномалии (Боллинджер)

Кнопка ⚡ Аномалии показывает доверительный коридор вокруг каждой серии:

  • Окно — размер скользящего окна для среднего и σ.
  • σ — множитель стандартного отклонения (обычно 2).

Точки, вылетающие за коридор, визуально выделяются.

T=0 — выравнивание по событию

Кнопка T=0, затем клик по графику — эта точка становится нулём. Ось X переключается в «секунды от T=0». Удобно для сравнения пусков/остановов между разными запусками.

Сбросить T=0 возвращает абсолютное время.

Уровни Lo / Hi

В строке каждого параметра есть поля Lo и Hi. Они рисуют пунктирные горизонтали и подсвечивают красным все сегменты, где значение вышло за пределы. Полезно для алармов, порогов, уставок.

Единицы измерения и конверсия

Из заголовка файла автоматически извлекается суффикс [Гц], [°C], [бар] и т.д. — он показывается над соответствующей осью Y цветом серии.

Поле Ед: в сайдбаре позволяет задать или изменить единицы вручную. Пусто = метка скрыта. В режиме Вместе метка выравнивается по правому краю оси (текст уходит от оси влево), в режиме Раздельно — по левому краю над осью.

Кнопка ⇌ рядом с полем «Ед:» появляется автоматически, если текущая единица поддерживает конверсию. Клик открывает меню с возможными целями. Клик по цели — пересчёт отображаемых значений (и Lo/Hi, если заданы) в новую единицу; исходные raw-значения сохраняются для точного возврата.

Поддерживаемые группы:

  • Температура: °C ↔ °F ↔ K
  • Давление: bar ↔ psi ↔ kPa ↔ MPa
  • Мощность: W ↔ kW ↔ MW (и Вт ↔ кВт ↔ МВт)
  • Длина: m ↔ ft ↔ km
  • Скорость: m/s ↔ km/h ↔ mph
После конверсии график, статистика и выровненные CSV используют выбранную единицу, а raw-long CSV дополнительно показывает исходные raw-значения и raw-unit.

Оси и высота графика

  • Высота — 0 = авто (заполняет вьюпорт), иначе фиксированная в пикселях.
  • Расст. осей Y — расстояние между несколькими Y-осями в режиме «Вместе», в пикселях (15–120 px). Расстояние не зависит от ширины окна.
  • Ось Y — ручной Min/Max для режима «Раздельно». Поддерживает запятую как десятичный разделитель.
  • Ось X — время — два слайдера «От/До» и поля чч:мм:сс для точной установки диапазона.
  • Текущий диапазон оси — сверху и снизу каждой оси Y всегда видны её текущие max и min. Значения обновляются живьём при зуме и при ручном редактировании границ (клик по экстремальным областям оси).

Экспорт

CSV по умолчанию сохраняется как UTF-8 с BOM, чтобы не терять символы единиц измерения вроде °C, µ, ±. Для старых Excel есть отдельный legacy-экспорт CP1251.

  • Разделитель колонок — ;.
  • Числа с запятой (23,145).
  • Время как datetime: ДД.ММ.ГГГГ чч:мм:сс.ммм (мс через точку — иначе Excel не распознаёт). После импорта: выдели весь диапазон → «Вставка → Диаграмма → Точечная с линиями» — получишь тот же график.
  • Legacy CP1251 нужен только для старых окружений; непредставимые Unicode-символы заменяются ASCII.
  • В заголовок колонки добавляются единицы в квадратных скобках, если они заданы: DWATT [Вт].
  • Если параметры покрывают разные временные окна — строки объединяются, в «пустых» участках у колонки стоит пробел (Excel рисует разрыв линии).

Все варианты собраны под единой кнопкой ⬇ Экспорт ▾ в шапке:

  • ⬇ CSV сырой long — реальные исходные точки без интерполяции: локальное время, epoch µs при наличии, источник времени, тег, значение, unit, status, merge-conflict, файл.
  • ⬇ CSV выровненный — wide-таблица по объединённой временной сетке; пустые промежутки не экстраполируются, но значения внутри диапазона интерполируются.
  • ⬇ CSV прореженный — выровненная таблица после текущего прореживания (до 5000/серия).
  • ⬇ CSV видимый — только текущий видимый диапазон оси X (после зума колёсиком или дёрганья миникарты).
  • ⬇ CSV сырой CP1251 — legacy-режим для старого Excel.
  • ⬇ PNG — PNG-рендер графика (1920 px, scale ×2). Добавляется название сверху. В режиме «Раздельно» склеивается в один файл с заголовками. Название и единицы разнесены по вертикали — в экспорте гарантированно не перекрываются.

Плюс:

  • 💾 рядом с именем файла (в шапке после переименования тегов) — скачать исходный файл с переименованными тегами.

Горячие клавиши

  • F1 — открыть/закрыть эту справку.
  • Esc — закрыть справку · отменить режим «+ Поставить» · убрать курсоры A/B.
  • [ / ] — прыжок к предыдущему/следующему видимому маркеру.
  • Alt+← / Alt+→ — история зума: откатить / повторить. Те же действия у кнопок ◀ ▶ в шапке. Стек на 40 переходов.
  • ← / → — двигать курсор B (в режиме «📏 Измерение»).
  • + / − или = — зум вокруг курсора B.
  • Shift+клик по строке параметра — solo-режим: только этот параметр.
  • Enter — зафиксировать значение в поле ввода (имя, единица, Lo/Hi).
  • Двойной клик по графику — сброс зума (autorange).

Формат исходного файла

Поддерживаются два варианта:

  • Широкий — общие колонки Дата, Время, мс (и опционально метка времени), далее значения. Используется в этом проекте.
  • Групповой — повторяющиеся группы по 5 колонок: Дата <тег>, Время <тег>, мс, status, значение.

Разделитель — табуляция. Десятичный — запятая (23,145) или точка. Единицы в заголовке пишутся в квадратных скобках: [Гц], [°C], [psi]. Первая строка может быть %PAHEADER% (пропускается). В grouped-формате поддерживаются русские Дата/Время и английские Date/Time заголовки.

Если есть epoch-колонка, она используется как источник времени. Если теги в двух файлах совпадают — их данные сливаются по времени, точные дубликаты убираются, а конфликтующие точки с одинаковым временем помечаются в диагностике и raw-long CSV. В шапке при этом показывается ⟷ N.

Что хранится между запусками

Всё локально, никаких внешних сервисов. Полные сессии сохраняются только по явной команде пользователя.

  • IndexedDB pagraph/sessions — полные сессии (данные + вид). При сохранении приложение просит у браузера persistent storage, если API доступен.
  • localStorage.loggraph_presets — пресеты вида (без данных).
  • localStorage.loggraph_markers — типизированные маркеры (переживают сброс данных).
  • localStorage.pagraph_sessions — устаревший ключ; если остался — автоматически мигрирует в IndexedDB при первом сохранении/загрузке.

Очистить всё разом: DevTools → Application → Storage → Clear site data. Или через Сброс (текущие данные; пресеты/сессии не трогает) и ручное удаление ✕.

Копия в другой браузер/машину: маркеры экспортируются JSON, сессии — через меню Сессия → Экспорт в файл.
